# МЕГА ИГРА (2001)

![Обложка](cover.png)

**Простой шуточный сайт выдуманного игрового журнала «МЕГА ИГРА» №10 (89), октябрь 2001 года.**

Сайт полностью статический, сделан в стилистике начала 2000-х (таблицы, яркие акценты, старые веб-приёмы) и идеально подходит для публикации на **GitHub Pages**.

- Письма читателей
- Много ретро-рекламы (GeForce 2, модемы 56k, CD с 1000 играми)
- Пасхалки (включая классический Konami Code)
- Полностью на русском языке

## Как запустить локально

Просто откройте `index.html` в любом браузере.

Или запустите локальный сервер:

```bash
python3 -m http.server 8000
# затем откройте http://localhost:8000
```

## Как выложить на GitHub Pages

1. Создайте новый репозиторий на GitHub (можно назвать `mega-igra-2001` или любое другое имя).

2. Загрузите файлы в репозиторий:
   ```bash
   git init
   git add .
   git commit -m "Первый номер МЕГА ИГРА — октябрь 2001"
   git branch -M main
   git remote add origin https://github.com/ВАШ_НИК/ИМЯ_РЕПО.git
   git push -u origin main
   ```

3. В настройках репозитория (Settings → Pages) выберите:
   - Source: **Deploy from a branch**
   - Branch: `main`
   - Folder: `/ (root)`

4. Через 1–2 минуты сайт будет доступен по адресу:
   `https://ВАШ_НИК.github.io/ИМЯ_РЕПО/`

Готово! Теперь все могут оценить, какой была «МЕГА ИГРА» в 2001-м.

## Подключение своего домена (mega-igra.ru)

Если хочешь, чтобы сайт открывался по адресу **mega-igra.ru** (а не по `твойник.github.io`):

### 1. В GitHub
1. Перейди в репозиторий → **Settings → Pages**.
2. В разделе **Custom domain** введи `mega-igra.ru` и нажми **Save**.
   - GitHub создаст (или уже создал) файл `CNAME` в корне репозитория с содержимым `mega-igra.ru`.
   - Зафиксируй этот файл (`git add CNAME && git commit -m "Add CNAME for custom domain" && git push`).

### 2. Настройка DNS (самое важное!)

Сейчас у домена `mega-igra.ru` **нет DNS-записей** (поэтому GitHub пишет "DNS check unsuccessful" и InvalidDNSError).

Зайди в панель управления доменом у своего регистратора (.ru обычно reg.ru, nic.ru, beget и т.д.) и добавь следующие записи:

**Для корневого домена (apex):**

| Тип | Имя / Host | Значение                  |
|-----|------------|---------------------------|
| A   | `@`        | `185.199.108.153`         |
| A   | `@`        | `185.199.109.153`         |
| A   | `@`        | `185.199.110.153`         |
| A   | `@`        | `185.199.111.153`         |

(Можно также добавить 4 AAAA записи для IPv6, если хочешь — они в официальной документации GitHub.)

**Для поддомена www:**

| Тип   | Имя / Host | Значение                  |
|-------|------------|---------------------------|
| CNAME | `www`      | `swrd-ru.github.io`       |

> **Важно**: 
> - Имя для корня обычно `@` или оставь поле пустым / mega-igra.ru.
> - Для CNAME в конце значения часто нужна точка: `swrd-ru.github.io.`
> - После сохранения DNS подожди 5–60 минут (иногда дольше для .ru зон).

### 3. Проверка
- Проверь на https://dnschecker.org (выбери A для mega-igra.ru и CNAME для www.mega-igra.ru).
- Дождись, когда в GitHub Pages Custom domain появится зелёная надпись **DNS check successful**.
- Поставь галочку **Enforce HTTPS** (может появиться не сразу, до 24 часов).

GitHub автоматически настроит редиректы между `mega-igra.ru` и `www.mega-igra.ru`.

### Альтернатива (рекомендую): Cloudflare
Если в панели регистратора неудобно добавлять 4 A-записи:
1. Зарегистрируйся на https://cloudflare.com (бесплатно).
2. Добавь домен и смени nameservers у регистратора на Cloudflare.
3. В Cloudflare добавь те же A и CNAME записи (можно включить Proxy — оранжевое облако).
4. Cloudflare даёт бесплатный SSL и проще управлять.

После этого в настройках GitHub тоже укажи `mega-igra.ru`.

Готово — сайт будет по-настоящему на mega-igra.ru как в 2001-м!
